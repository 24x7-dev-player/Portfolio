Cloud Computing:

  • What is Cloud Computing?

    Cloud computing is the on-demand delivery of IT resources over the Internet with pay-as-you-go pricing. It allows access to technology services like computing power, storage, and databases from providers like AWS.

  • Key Features of Traditional Computing:
    • Control: Complete control over infrastructure.
    • Security: Responsible for securing data and systems.
    • Scalability: Expensive and time-consuming to expand.

Key Features:

  • Scalability: Easily adjust resources without buying more hardware, adapting quickly to changes in demand.
  • Cost-Efficiency: Pay-as-you-go model eliminates large upfront investments and reduces maintenance costs.
  • Accessibility and Collaboration: Access applications and data from anywhere, promoting teamwork and productivity.
  • Security and Reliability: Leading providers prioritize security with encryption, backups, and disaster recovery plans.
  • Application Development Efficiency: Offers a flexible, scalable platform for optimizing the application development lifecycle.
  • Rapid Provisioning and Scalability: Quick provisioning of resources, enabling easy scaling for optimal performance.
  • Cost-Effectiveness: Pay only for used resources, eliminating upfront infrastructure investments and saving on maintenance.
  • Collaborative Development: Facilitates real-time collaboration among team members, enhancing productivity.
  • Focus on Development: Offloads infrastructure management, allowing teams to focus on coding, testing, and deployment.

Infrastructure As A Service:

  • Infrastructure As A Service (IaaS):
    • Flexible cloud service, rents hardware.
    • Control over servers, VMs, storage, networks, OS.
    • Instant, managed computing infrastructure.
    • Examples: AWS EC2, DigitalOcean, Azure, Google Compute Engine.
  • Use-Cases:
    • Data backup, storage, and recovery.
    • Cost-effective test and development environments.
    • Increased computing power for Big Data.
    • Scalable for complex web projects.
    • Handles tasks requiring supercomputers.
    • Run graphic-intensive apps without latency.
  • Disadvantages:
    • Limited infrastructure control.
    • Users responsible for security.
    • Possible restricted access in some regions.

Platform As A Service:

  • Platform As A Service (PaaS):
    • Ready-to-use development environment.
    • Focus on code without managing infrastructure.
    • Scalable and feature-rich services.
    • More expensive than IaaS, cheaper than SaaS.
    • Examples: AWS Elastic Beanstalk, Azure WebApps, Google Cloud SQL.
  • Benefits:
    • Faster development time.
    • Reduced costs on manual deployments.
    • Enhanced security for applications.
    • High availability even during failures.
  • Use-Cases:
    • Developing and managing APIs and microservices.
    • Setting up and managing databases.
  • Disadvantages:
    • Limited infrastructure control.
    • Dependency on provider for maintenance.
    • Restricted flexibility for specific workloads.

Software As A Service:

  • Software As A Service (SaaS):
    • Complete product run and managed by the provider.
    • Hosted online and available via subscription or purchase.
    • No need to worry about maintenance or infrastructure management.
    • Examples: Microsoft Office 365, Oracle ERP/HCM Cloud, SalesForce, Gmail, Dropbox.
  • Benefits:
    • Lower Total Cost of Ownership (TCO).
    • Eliminates hardware expenses and maintenance costs.
    • Improved security with secure servers and 24/7 monitoring.
  • Disadvantages:
    • Limited customization options.
    • Dependency on internet connectivity.
    • Potential security incidents and data breaches.
    • Limited control over data.

Cloud Deployment:

  • Public Cloud: Available to all, managed by providers. Examples: Amazon EC2, Google App Engine.
  • Private Cloud: Used by one organization, not shared. Examples: AWS, Microsoft Azure.
  • Hybrid Cloud: Combination of public and private clouds, for smooth transition.

EC2 Commands:

  • Connect to EC2 Instance:
    • SSH into your instance using: 
      ssh -i /path/to/your-file.pem ubuntu@ec2-xx-xx-xx-xx.compute-1.amazonaws.com
    • Replace /path/to/your-file.pem with your actual .pem file path and ec2-xx-xx-xx-xx.compute-1.amazonaws.com with your instance's public DNS or IP address.
  • Set .pem File Permissions:
    • Ensure your .pem file has read-only permissions: 
      chmod 400 /path/to/your-file.pem
  • Check System Information:
    • CPU Info: 
      lscpu
    • RAM Info: 
      free -h
    • OS Info: 
      cat /etc/os-release
    • Disk Usage: 
      df -h
