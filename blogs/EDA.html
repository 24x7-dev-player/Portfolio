Exploratory Data Analysis

  • Understanding Errors:
    • False Positive (FP): Model predicts positive class (1) for a negative case (0) (e.g., medical test saying you have disease when you don't).
    • False Negative (FN): Model predicts negative class (0) for a positive case (1) (e.g., medical test saying you're healthy but you have a disease).
  • Dealing with Missing Data: Many missing values require handling before building models (imputation, deletion, etc.).
  • Project Considerations:
    • No urgency for fast predictions allows focusing on accuracy over speed.
    • Interpretability unimportant: complex models like deep learning are suitable.
    • Misclassification cost: high cost associated with both false positives and negatives (e.g., unnecessary repairs).

KNN imputer

  • KNN Imputer: Fills missing values using k-nearest neighbors' feature values.
  • Scalers (for normalizing features):
    • StandardScaler (good for normal distributions): Removes mean and scales to unit variance (sensitive to outliers).
    • RobustScaler (good for outliers or skewed data): Uses median and IQR for scaling (less sensitive to outliers than StandardScaler).
    • MinMaxScaler (scales to a range, often 0-1): Preserves distribution shape but sensitive to outliers (uses min and max values).
  • Oversampling and Undersampling Techniques (for imbalanced datasets):
    • SMOTE (Synthetic Minority Over-sampling Technique): Creates synthetic samples for the minority class using k-nearest neighbors.
    • Tomek Links (removes noisy instances): Identifies and removes instance pairs from different classes that are very close (potentially noisy or misclassified).
    • SMOTE+TOMEK (combination): Oversamples minority with SMOTE first, then removes noisy instances from both classes using Tomek Links.
