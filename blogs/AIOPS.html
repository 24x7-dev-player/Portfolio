Github:

  • Repo: Project files & history (`git init`)
  • Commit: Snapshot of changes (`git commit -m "message"`)
  • Branch: Separate development line (`git branch new-branch-name`)
  • Merge: Combine changes from branches (`git merge branch-to-merge`)
  • Remote: Shared repo for collaboration (`git remote add origin `)
  • Pull: Get changes from remote (`git pull origin main`)
  • Push: Send changes to remote (`git push origin main`)
  • Clone: Copy remote repo (`git clone `)
  • Status: Check repo state (`git status`)
  • Log: View commit history (`git log`)
  • Branch: Create git branch, Switch git checkout, (Create & Switch) git checkout -b, Delete git branch -d, Rename git branch -m
  • Stage: Add file git add, Add all git add ., Remove file git reset, Unstage all git reset
  • Compare: Changes git diff, Staged git diff --staged
  • Undo: Discard file git checkout --, Discard all git reset --hard, Undo commit git reset HEAD~, (Undo & Discard) git reset --hard HEAD~
  • History: View git log, (Short) git log --oneline, Details git show
  • Remote: Fetch git fetch, Pull git pull, Push git push
  • Rebase/Merge: Choose wisely! Fetch git fetch origin, Rebase git rebase origin/main, Merge git merge origin/main && git push origin main

Docker:

  • docker build -t <imagename> .: Build a Docker image with the specified name.
  • docker run -d -p 5000:5000 helloapp: Run a Docker container in detached mode with port mapping.
  • docker login: Log in to a Docker registry.
  • docker tag myhelloapp username/reponame or imagename: Tag a Docker image with a new name.
  • docker push username/reponame or imagename: Push a Docker image to a registry/repository.
  • docker run -it ubuntu /bin/bash: Run a new Docker container interactively with a Bash shell.
  • docker images: List all Docker images.
  • docker ps: List all running Docker containers.
  • docker ps -a: List all Docker containers, including those that are not running.
  • docker rmi -f jenkins/jenkins: Remove a Docker image.
  • docker rm -f jenkins/jenkins: Remove a Docker container.
  • docker run -p 8080:8080 jenkins/jenkins:lts: Run a Docker container with port mapping.
  • FROM ubuntu: Specify the base image as Ubuntu in a Dockerfile.
  • WORKDIR /tmp: Set the working directory inside the Docker container.
  • RUN echo "Subscribe the ineuron intelligence": Execute a command during the image build process.
  • ENV myname sunny: Set an environment variable inside the Docker container.
  • COPY testfile /tmp: Copy a file into the Docker container.
  • ADD test.tar.gz /tmp: Add and extract an archive into the Docker container.
